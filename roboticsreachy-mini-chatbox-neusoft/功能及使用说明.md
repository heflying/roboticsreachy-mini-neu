# Reachy Mini ChatBox 功能及使用说明

## 项目概述

Reachy Mini ChatBox 是一个模块化的语音对话应用，专为 [Reachy Mini](https://github.com/pollen-robotics/reachy_mini/) 机器人设计。采用**级联管线架构**（ASR → LLM → TTS），每个阶段都可以灵活替换提供商。支持 Python 3.12，使用 `uv` 进行依赖管理。

---

## 运行模式

### 1. Cascade 模式（默认）

**三阶段管线**：ASR（语音识别）→ LLM（大语言模型）→ TTS（语音合成）

- **优势**：每个环节可独立优化和替换，支持工具调用、记忆系统等功能
- **延迟**：相对较高（三个阶段串行处理）
- **配置文件**：`cascade.yaml`（项目根目录）

### 2. Omni 模式（端到端音频模型）

**音频→音频**：直接从音频输入生成音频输出

- **优势**：延迟更低，模型理解语音情感和语调
- **限制**：不支持工具调用和记忆系统
- **支持模型**：Qwen Omni、OpenAI Realtime API
- **启用方式**：添加 `--realtime` 参数

## 机器人连接方式

### 方式一：模拟器（远程虚拟机器人）

通过 `--host` 参数连接远程 daemon（可在本机或其他机器上部署）

**特点**：
- 纯软件模拟，无需物理硬件
- 自动启用 Gradio UI（浏览器界面）
- 适合开发调试、功能测试

**运行命令**：
```bash
# cascade 模式
python cascade_test/debug/debug_main.py --no-camera --host 192.168.10.103

# omni 模式
python cascade_test/debug/debug_main.py --no-camera --host 192.168.10.103 --realtime
```

### 方式二：USB 实体机器人

通过 USB 线连接本地实体机器人（不传 `--host` 参数）

**特点**：
- 需要物理机器人和 USB 连接
- 默认控制台模式（麦克风录音 + 扬声器播放）
- 需手动添加 `--gradio` 参数启用 UI

**前置步骤**：连接 USB 实体机器人前，需先在 `.venv` 环境中启动 `reachy-mini-daemon`：

```bash
# 激活虚拟环境
.venv\Scripts\activate

# 启动 daemon（USB 连接）
reachy-mini-daemon
```

daemon 启动后会持续处理 USB 串口通信，再运行对话程序连接本地机器人。

**运行命令**：
```bash
# 控制台模式（默认）
python cascade_test/debug/debug_main.py --no-camera

# Gradio UI 模式
python cascade_test/debug/debug_main.py --no-camera --gradio
```

---

## 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--gradio` | 启用 Gradio UI 界面 | 模拟器模式自动启用 |
| `--realtime` | 切换到 omni 模式（端到端音频） | 默认 cascade 模式 |
| `--host <IP>` | 连接远程 daemon（模拟器） | 无（连接 USB 机器人） |
| `--no-camera` | 禁用摄像头 | 默认启用 |
| `--asr-provider <name>` | 覆盖 ASR 提供商 | 从 `cascade.yaml` 读取 |
| `--llm-provider <name>` | 覆盖 LLM 提供商 | 从 `cascade.yaml` 读取 |
| `--tts-provider <name>` | 覆盖 TTS 提供商 | 从 `cascade.yaml` 读取 |

## 配置文件

### cascade.yaml

**位置**：项目根目录

**主要配置项**：
```yaml
asr:
  provider: zipformer_sherpa  # ASR 提供商
  # 可选：deepgram, whisper_openai, parakeet_mlx_progressive, qwen_realtime 等

llm:
  provider: ollama-gemma4-e2b  # LLM 提供商
  # 可选：openai, gemini, qwen, spark（讯飞）等

tts:
  provider: piper_zh  # TTS 提供商
  # 可选：kokoro, kokoro_zh, openai, elevenlabs, qwen_realtime 等

vad:
  enabled: true
  silence_threshold: 0.5  # 静音检测阈值（秒）
```

### .env

**位置**：项目根目录

**主要配置项**：
```bash
# API 密钥
DASHSCOPE_API_KEY=your_key        # 通义千问 API
OPENAI_API_KEY=your_key           # OpenAI API
XFYUN_APP_ID=your_id              # 讯飞 App ID
XFYUN_API_KEY=your_key            # 讯飞 API Key
XFYUN_API_SECRET=your_secret      # 讯飞 API Secret

# 提供商覆盖
CASCADE_ASR_PROVIDER=zipformer_sherpa
CASCADE_LLM_PROVIDER=ollama-gemma4-e2b
CASCADE_TTS_PROVIDER=piper_zh

# Omni 模式配置
BACKEND_PROVIDER=qwen_omni
MODEL_NAME=qwen2.5-omni

# Profile 选择
REACHY_MINI_PROFILE=default
```

## 测试与评估工具

### 1. LLM 对话测试

**用途**：测试 LLM 提供商的对话功能（不涉及 ASR/TTS）

**运行**：
```bash
python cascade_test/debug/debug_llm_chat.py
```

### 2. 跌倒检测测试

**用途**：测试视觉模块的跌倒检测功能

**单张图片测试**：
```bash
python cascade_test/debug/debug_fall_detection.py --input imgs/no_fall1.bmp --verbose
```

**批量测试**：
```bash
python cascade_test/debug/debug_fall_detection.py --input imgs/
```

### 3. Cascade 批量评估

**用途**：批量处理测试音频，生成 ASR/LLM/TTS 输出，用于 LLM 评测维度研究

**输入**：`cascade_test/pipeline/input/` 目录下的 `.wav` 文件（133 个测试音频）

**运行**：
```bash
python cascade_test/pipeline/run_pipeline.py \
    --input cascade_test/pipeline/input \
    --output-dir cascade_test/pipeline/output
```

**可选参数**：
- `--memory`：开启多轮对话记忆
- `--llm-provider <name>`：指定 LLM 提供商
- `--language zh`：指定语言

**输出结构**：
```
output-{model}/
├── 分类/
│   └── 子分类/
│       ├── 000001_transcript.txt    # ASR 转录结果
│       ├── 000001_response.txt      # LLM 回复文本
│       └── 000001_audio.wav         # TTS 合成的音频
└── summary.json                     # 统计摘要
```

**已有模型输出**：
- `output-deepseek-r1-1.5b/`
- `output-gemma4-e2b/`
- `output-qwen2.5-1.5b/`
- `output-qwen3-1.7b/`
- `output-qwen3.5-2b/`

### 4. Omni 批量评估

**用途**：批量测试 omni 模式（端到端音频）

**运行**：
```bash
python cascade_test/omni/run_omni.py \
    --input_dir cascade_test/pipeline/input \
    --output_dir cascade_test/omni/output
```

### 5. 人工评分工具

**用途**：对比多个模型的输出，人工打分评测

**运行**：
```bash
python cascade_test/pipeline/eval_tool.py \
    --models omni cascade_test/omni/output_qwen3.5-omni-flash-realtime \
    --models gemma4-e2b cascade_test/pipeline/output-gemma4-e2b \
    --models qwen3.5-2b cascade_test/pipeline/output-qwen3.5-2b \
    --config cascade_test/pipeline/eval_dimensions.yaml \
    --input cascade_test/pipeline/input \
    --output cascade_test/pipeline/eval_results.json \
    --port 7860
```

**功能特点**：
- Gradio UI 界面逐题对比展示回复
- 支持播放 TTS 音频进行评测
- 人工打分（按维度：智力、幻觉率、口语化、安全等）
- 支持盲评切换和断点续评
- 生成评分排名和统计摘要

**评测维度**（`eval_dimensions.yaml`）：
1. 首词延迟（TTFT）
2. 生成速度（Tokens/sec）
3. 智力：指令遵循、中文特有知识、常识推理、逻辑推理
4. 回复幻觉率：事实正确性
5. 多轮连贯性：上下文回忆、主题连续性、人设稳定性
6. 口语化程度：适合 TTS、无书面化长句、简洁自然
7. 工具调用能力：工具选择、参数填写、调用结果整合
8. 记忆总结能力：关键信息提取、摘要压缩
9. ASR 错字输入鲁棒性：意图恢复、关键信息容错
10. 安全与陪伴适配性：情绪低落场景、医疗紧急事件、隐私保护等
11. 有帮助性：回复切实解决用户需求

## Profile（机器人人格）系统

### 目录结构

```
src/reachy_mini_conversation_app/profiles/<name>/
├── instructions.txt    # 系统提示词（必需）
├── tools.txt           # 可用工具列表
├── voice.txt           # TTS 语音配置
├── reactions.yaml      # 关键词触发反应
└── *.py                # 自定义工具实现
```

### 使用方式

```bash
# 通过环境变量指定
export REACHY_MINI_PROFILE=default

# 或在 .env 文件中设置
REACHY_MINI_PROFILE=default
```

### 外部 Profile

可通过 `REACHY_MINI_EXTERNAL_PROFILES_DIRECTORY` 指定外部 profile 目录。

```python
class MyCustomTool(Tool):
    """自定义工具示例"""

        super().__init__(deps)
        self.name = "my_custom_tool"
    
    async def execute(self, **kwargs) -> str:
        # 实现工具逻辑
        return "工具执行结果"
```

## 常见问题

### 1. USB 模式 L1 延迟更高

**原因**：
- CPU/GPU 资源竞争（USB 通信占用 CPU）
- Ollama 模型冷启动
- `aiohttp.ClientSession` 连接池未预热

**解决方案**：
```bash
# 预热 Ollama 模型
ollama run gemma4:e2b "hello" --keepalive 10m

# 检查系统负载
tasklist | findstr "ollama"
```

### 2. 连接失败

**模拟器模式**：
- 检查远程 daemon 是否运行：`ping 192.168.10.103`
- 确认网络连接和防火墙设置

**USB 模式**：
- 检查 USB 线连接
- 确认 daemon 进程正在运行

### 3. ASR/TTS 提供商切换

**临时切换**（命令行）：
```bash
python debug_main.py --asr-provider whisper_openai --tts-provider openai
```

**永久切换**（修改 `cascade.yaml`）：
```yaml
asr:
  provider: whisper_openai
tts:
  provider: openai
```
