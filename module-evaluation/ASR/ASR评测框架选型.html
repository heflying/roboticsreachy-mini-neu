ASR 评测框架选型报告

基于「表4 最终评测指标」的 9 大类、29 项指标需求

一、评测体系需要什么能力

需求层做什么对应指标
音频条件构造 在干净测试音频上叠加噪声、变速、加混响等 噪声鲁棒性、语速鲁棒性、远场退化
ASR 推理 把音频转成文本（支持流式，用于实时性测量） 所有指标的上游输入
文本对齐与评分 比对参考文本 vs 识别文本，算 CER 并拆解 Sub/Del/Ins/Hits 准确率、诊断
时间打点 记录推理关键时刻的时间戳 RTF / TTFC / 收敛延迟 / E2E Latency
核心发现：没有单一框架能覆盖这四层。评测体系天然跨域（音频处理 + 模型推理 + 文本评分 + 时间测量），必须分层选工具。

二、候选框架全景

2.1 文本评分层

框架CERSub/Del/Ins中文友好集成难度评判
jiwer ✅✅ 原生拆分✅ 逐字分词即可极低（pip install） 选用
ESPnet eval ✅⚠️ 需自己拆✅高（依赖 ESPnet 生态） 不选
SpeechBrain ✅⚠️ 需自己拆✅高 不选
自定义编辑距离 ✅✅✅中等（造轮子） 不选

选择 jiwer 的理由：唯一一个「pip install 即用 + 原生输出 Sub/Del/Ins/Hits 拆分」的库。底层调用 RapidFuzz（C++ 编辑距离），速度快。中文只需按字符切分即可。

框架加噪声变速远场模拟SNR 控制评判

librosa ⚠️ 需手写✅ time_stretch❌⚠️ 手动 不选
pydub ⚠️ 基础叠加✅ 变速❌❌ 无精确 SNR 不选
pedalboard ✅ 音频效果❌❌❌ 不选

选择 audiomentations 的理由：唯一一个同时覆盖「任意噪声叠加 + 精确 SNR 控制 + 变速 + RIR 远场模拟」的库。API 与图像增强库 albumentations 一致，一个 transform pipeline 即可完成多条件构造。

2.3 ASR 推理层

框架流式支持多模型ONNX/无GPUPython API评判
sherpa-onnx ✅ 原生流式✅ Paraformer/SenseVoice/Whisper 等✅ ONNX，CPU 可跑✅ 选用
FunASR ⚠️ 需额外配置✅ 丰富⚠️ 部分模型需 GPU✅ 不选
Whisper (openai) ❌ 非流式❌ 单一⚠️ 较大✅ 不选
ESPnet ⚠️ 仅部分模型✅❌ 需 GPU✅ 不选
faster-whisper ❌ 非流式❌ 单一✅ CTranslate2✅ 不选

选择 sherpa-onnx 的理由：原生流式推理是核心——评测需要 TTFC 和收敛延迟，非流式 ASR 根本测不了这两个指标。ONNX 格式确保评测环境灵活（有无 GPU 均可）。同时支持 Paraformer（中文最强之一）和 SenseVoice（带情感/事件检测）等多模型，便于横向对比。

2.4 实时性打点

无现成框架，需要基于 sherpa-onnx 的流式 API 自定义脚本。

大约 80~120 行 Python：hook 推理开始/结束时间戳，记录流式回调中首次文本到达和文本 hash 收敛的时刻。

五、为什么排除其他热门框架

框架排除原因
FunASR 定位是推理框架而非评测框架。它能跑多种 ASR 模型，但评测能力薄弱（无原生 Sub/Del/Ins 拆分，无噪声构造）。与 sherpa-onnx 功能重叠但 sherpa-onnx 流式支持更好、依赖更轻。
ESPnet 学术级全链路工具，功能强大但太重。安装需配置 Kaldi 依赖，主要面向模型训练而非评测。评测脚本是其 recipe 的一部分，难以独立使用。
SpeechBrain 同 ESPnet，面向训练而非评测。需 PyTorch + GPU，与轻量评测场景不匹配。
Whisper / faster-whisper 不支持流式推理。无法测量 TTFC 和收敛延迟——这两个是养老机器人场景的关键实时性指标。
EVAL-ASR (IEEE 2025) 论文刚发表，开源代码尚未确认。理念与我们的评测体系最接近（多维度中文 ASR 评估），待其开源后可作为参考框架叠加。

✅ 最终选型结论

1
audiomentations — 音频条件构造
唯一同时覆盖噪声叠加（精确 SNR）、变速、远场模拟的工具。pip install 即可。
2
sherpa-onnx — ASR 推理引擎
原生流式推理是选择它的核心原因（非流式无法测 TTFC/收敛延迟）。ONNX 格式，CPU 可跑，支持多模型横向对比。
3
jiwer — 文本对齐与评分
唯一轻量级输出 CER + Sub/Del/Ins/Hits 完整诊断拆解的库。底层 C++ 编辑距离，快且准。
4
自定义打点脚本 — 实时性测量
基于 sherpa-onnx 流式回调，约 80~120 行 Python，测量 RTF / TTFC / 收敛延迟 / E2E Latency。

原则：每个领域选该领域最专精的工具，不做大而全的框架绑定。 三个库加起来 pip install audiomentations sherpa-onnx jiwer 一行命令即可安装完毕。
