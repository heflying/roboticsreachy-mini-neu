# eval_jiwer — ASR 评测管线 功能及使用说明

## 项目定位

对多种 Sherpa-ONNX 流式 ASR 模型进行标准化的 **准确率**（CER/Sub/Del/Ins）和 **实时性**（RTF/TTFC/E2E）评测，并生成 HTML 可视化对比报告。

## 核心功能

### 1. 评测管线 (`run_eval.py`)

对指定模型在指定数据集上进行完整的流式 ASR 评测。

**工作流程：**

```
加载模型 → 加载数据集 → 对每条话语:
  1) VAD 切句（Silero VAD）
  2) 逐句 ASR 流式推理（200ms chunk）
  3) TimingTracker 实时性打点
  4) 拼接全对话文本 → 与标注计算 CER (jiwer)
→ 聚合指标 → 输出 JSON
```

**用法：**

```powershell
cd eval_jiwer
.venv\Scripts\python run_eval.py `
    --model-type sherpa_onnx_zipformer `
    --sherpa-model-dir models/ASR/zipformer-zh `
    --manifest manifests/seniortalk_test.jsonl manifests/xiaoya_gen.jsonl `
    --output-dir results
```

**常用参数：**

| 参数 | 说明 |
|------|------|
| `--model-type` | `sherpa_onnx_zipformer` 或 `sherpa_onnx_paraformer` |
| `--sherpa-model-dir` | 模型目录路径 |
| `--manifest` | 数据集 JSONL 文件，支持多个 |
| `--max-utterances N` | 限制评测条数（调试用） |
| `--max-hours N` | 限制音频总时长 |
| `--keep-punctuation` | 保留标点参与 CER 计算 |
| `--vad-threshold` | VAD 语音检测阈值（默认 0.5） |
| `--vad-min-speech-ms` | 最小语音段长度 ms（默认 250） |
| `--vad-min-silence-ms` | 最小静音段长度 ms（默认 300） |
| `--dry-run` | 仅统计不推理 |
| `--verbose` | 详细日志 |

**输出：** `results/{model_name}_{timestamp}.json`

---

### 2. 对比报告生成 (`compare_results.py`)

扫描多个模型的评测结果 JSON，生成可视化 HTML 对比报告。

**用法：**

```powershell
cd eval_jiwer
.venv\Scripts\python compare_results.py              # 自动扫描 results/cmp_*.json
.venv\Scripts\python compare_results.py --open       # 生成后自动打开浏览器
.venv\Scripts\python compare_results.py -o my_report.html  # 指定输出路径
```

**报告包含：**

- **指标总览卡片** — CER、Sub/Del/Ins 率、SER、RTF、TTFC 等跨数据集加权汇总
- **硬件环境表** — CPU 型号、核心数、内存等
- **CER 详细对比表** — 各模型在各数据集上的 CER/Sub/Del/Ins/Hits/RefLen/句数
- **实时性对比表** — RTF、TTFC（avg/P50/P90/P95/P99）、E2E 延迟
- **评测配置表** — VAD 参数、数据量限制等
- **指标说明** — 所有指标的含义解释

## 评测指标说明

### 准确率 / 错误率

| 指标 | 含义 | 计算方式 |
|------|------|---------|
| **CER** | 字符错误率 | (Sub + Del + Ins) / RefLen，越低越好 |
| **Accuracy** | 字符正确率 | Hits / RefLen，越高越好 |
| **SER** | 句错率 | CER > 1% 的句子占比，越低越好 |
| **Sub 率** | 替换错误率 | 被替换字符数 / RefLen |
| **Del 率** | 删除错误率 | 漏识别字符数 / RefLen |
| **Ins 率** | 插入错误率 | 幻觉多余字符数 / RefLen |
| **Hits** | 正确匹配字符数 | — |
| **RefLen** | 参考文本总字数 | — |

### 实时性

| 指标 | 含义 | 说明 |
|------|------|------|
| **RTF** | 实时率 | 处理时长 / 音频时长，< 1 表示快于实时 |
| **TTFC** | 首字延迟 | 首个 chunk 到首次出字的时间 |
| **P50/P90/P95/P99 TTFC** | TTFC 百分位数 | 评估长尾延迟 |
| **E2E Latency** | 端到端延迟 | 首个 chunk 到最终完整文本的时间 |

## VS Code 运行配置

项目根目录 `launch.example.json` 提供了 3 个预配置的调试条目，可直接按 F5 启动：

| 条目名称 | 功能 |
|---------|------|
| `Python Debugger: eval` | 运行评测管线 `run_eval.py`，对指定模型在 SeniorTalk + 小雅数据集上评测。默认使用 zipformer-transducer-zh-en-int8 模型，其他模型可取消注释切换。 |
| `Python Debugger: compare` | 运行 `compare_results.py`，扫描 `results/cmp_*.json` 生成对比报告 HTML。 |
| `Python Debugger: simple_test_input_file` | 运行 `asr_test.py`，对单个音频文件快速测试 ASR 效果。 |

所有条目均使用 `eval_jiwer/.venv` 虚拟环境，工作目录为 `eval_jiwer/`。
