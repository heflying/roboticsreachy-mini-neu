三、逐一排除理由

模型 架构 年代 判定 排除原因
LSTM Transducer LSTM RNN-T ~2019 排除 架构已淘汰：三代以前的架构，已被 Conformer / Zipformer 全面超越。类比：用 ResNet-18 对比 ResNet-152，没有价值。
Conformer-zh Conformer + Transducer 2023.05 排除 已被 Zipformer 取代：Zipformer 是 K2-FSA 团队基于 Conformer 的正统升级版，相同团队、相同数据，测 Conformer 不如直接测 Zipformer。（Zipformer 论文已验证对 Conformer 的全面优势）
SenseVoice-Small 通义 CTC 2024.07 排除 非流式 + 精度不如 Paraformer：SenseVoice 核心卖点是情感识别和音频事件检测，纯 ASR 精度（CER ~3%）低于 Paraformer（1.95%）；且不支持流式解码，不符合 robot streaming 场景要求。
FireRedASR AED / LLM 2025.02 排除 需GPU + 非流式：参数 1.1B / 8.3B，CER 确实最低（AISHELL-1 0.57%），但需要 GPU 推理且非流式。机器人 CPU Only 场景根本跑不动。
Whisper (large-v3) Encoder-Decoder 2023 排除 中文精度显著落后：作为通用多语言模型，AISHELL-1 CER 约 5.14%，比国产中文专精模型差 3–5 倍。不是中文场景的最优选择。
四种排除原因总结：
🗑️ 架构已淘汰（不如 Zipformer）→ LSTM、Conformer
🔒 非流式（不符合 streaming 要求）→ SenseVoice、FireRedASR
🖥️ CPU 跑不动（需要 GPU）→ FireRedASR
🇨🇳 中文精度差 → Whisper

四、最终选定：2 个模型

Zipformer-zh
K2-FSA 自研 · Kaldi 路线天花板
参数量：~80 MB
架构：Zipformer + Transducer
流式：✅ 流式
CPU推理：✅ 可运行
定位：基线模型（当前代码已集成）
Paraformer-zh
达摩院出品 · FunASR 生态天花板
参数量：220 MB
架构：U2++ (CTC + Attention)
流式：✅ 流式
CPU推理：✅ 可运行
AISHELL-1 CER：1.95%
定位：精度天花板
最终测试矩阵：Zipformer + Paraformer
  • Zipformer-zh：基线模型，K2-FSA 路线代表，轻量 80MB，流式
  • Paraformer-zh：精度天花板，达摩院路线代表，220MB，AISHELL-1 CER 1.95%，流式
  • 两个模型覆盖 Kaldi 路线与达摩院路线两条技术路线
  • 在两个数据集上评测：AISHELL-1（普通场景）+ SeniorTalk（老年人场景）
  • 指标：WER / CER / RTF / TTFT
