# TTS 评测工具 — 功能及使用说明

## 项目概述

本项目用于养老陪伴机器人场景下 TTS（文本转语音）模型的评测，涵盖多款中文 TTS 模型的合成速度、首词延迟（TTFT）、实时率（RTF）等指标的测量与对比。

## 目录结构

```
eval_custom/
├── .env                  # 运行时配置（模型选择、语速等）
├── models.toml           # 模型参数定义（所有支持的 TTS 模型）
├── models/TTS/           # 模型文件（.onnx、tokens、lexicon 等）
├── text_files/           # 批量合成用的文本文件
├── output/               # batch_synthesize 的输出目录
├── result/               # test_streaming_tts 的输出目录
├── result_piper/         # test_piper_tts 的输出目录（按模型名分子目录）
├── streaming_tts.py      # sherpa-onnx 流式 TTS 核心模块
├── piper_streaming_tts.py # Piper 流式 TTS 核心模块
├── test_streaming_tts.py  # sherpa-onnx 模型测试脚本
├── test_piper_tts.py      # Piper 模型专用测试脚本
├── compare_results.py     # 模型评测结果对比脚本
├── batch_synthesize.py    # 批量合成脚本
└── .venv/                 # Python 虚拟环境
```

## 配置文件

### .env — 运行时配置

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `TTS_MODEL` | 当前使用的模型名称（必须在 `models.toml` 中有对应条目） | `piper-huayan` |
| `TTS_SPEED` | 语速倍率（可选，覆盖 models.toml 默认值） | 1.0 |
| `TTS_SPEAKER_ID` | 说话人 ID（可选） | 0 |
| `TTS_OUTPUT_DIR` | 输出目录名 | `result` |

### models.toml — 模型参数

定义所有支持的 TTS 模型及其路径、参数。每个模型是一个 TOML 表，关键字段：

| 字段 | 说明 |
|------|------|
| `type` | 模型类型：`piper` / `vits` / `kokoro` / `matcha` |
| `model` | ONNX 模型文件路径 |
| `tokens` | token 表路径 |
| `lexicon` | 词典路径 |
| `rule_fsts` | 规则 FST 路径（日期、数字、电话等） |
| `speaker_id` | 说话人编号 |
| `speed` | 默认语速 |
| `sample_rate` | 采样率 |

支持的模型列表：

- **Piper**: `piper-chaowen`、`piper-huayan`、`piper-xiao_ya`
- **VITS**: `vits-icefall-aishell3`、`vits-aishell3`、`vits-melo-zh-en`、`vits-hf-fanchen-C`、`vits-hf-theresa`、`vits-hf-eula`
- **Kokoro**: `kokoro-v1_0`、`kokoro-v1_1`
- **Matcha**: `matcha-icefall-zh-baker`

## 脚本功能详解

### 1. test_streaming_tts.py — sherpa-onnx 模型测试

**功能**：对句子列表逐条合成，实时播放音频，保存 wav 文件，生成统计报告。

**用法**：
```bash
cd eval_custom
python test_streaming_tts.py                              # 使用 .env 中的模型
python test_streaming_tts.py --model piper-chaowen        # 指定模型
python test_streaming_tts.py --listen                     # 实时播放
python test_streaming_tts.py --sentences-file my_sentences.txt  # 自定义文本
```

**输出**：`result/<模型名>/` 目录下：
- `01_好的没问题.wav` 等音频文件
- `stats.json` 统计数据（TTFT、RTF、合成时间等）

### 2. test_piper_tts.py — Piper 模型专用测试

**功能**：与 `test_streaming_tts.py` 逻辑一致，但专为 Piper 模型优化，模型名从 `.env` 的 `TTS_MODEL` 读取。

**用法**：
```bash
cd eval_custom
python test_piper_tts.py    # 使用 .env 中配置的 Piper 模型
```

**输出**：`result_piper/<模型名>/` 目录下：
- wav 音频文件
- `stats.json` 统计数据

### 3. compare_results.py — 模型评测结果对比

**功能**：读取各模型结果目录中的 `stats.json`，生成 HTML 对比报告，便于横向比较不同模型的 TTFT、RTF 等指标。

**用法**：
```bash
cd eval_custom
python compare_results.py                                       # 对比 result/ 目录
python compare_results.py --result-dir result_piper             # 对比 Piper 结果
python compare_results.py --output comparison_piper.html        # 自定义输出文件名
```

**输出**：HTML 对比报告，包含各模型的汇总统计表格。

### 4. batch_synthesize.py — 批量合成

**功能**：读取 txt 文件列表，将每一行文本转换为语音 wav 文件，编号保存。适用于大批量音频素材生成。

**用法**：
```bash
cd eval_custom
python batch_synthesize.py
```

**输出**：`output/` 目录下：
- `000001.wav`、`000002.wav` 等编号音频文件

**配置**（在脚本顶部修改）：
| 变量 | 说明 | 默认值 |
|------|------|--------|
| `TXT_FILES` | 输入文本文件列表 | `["text_files/1.txt"]` |
| `START` | 起始编号 | 1 |
| `OUTPUT_DIR` | 输出目录 | `output` |

## 评测指标说明

| 指标 | 英文 | 含义 |
|------|------|------|
| TTFT | Time To First Token | 首词延迟：从发起请求到第一个音频数据返回的时间 |
| RTF | Real Time Factor | 实时率：合成时间 / 音频时长，值越小越好（<1 表示快于实时） |
| 合成时间 | Synthesis Time | TTS 模型推理耗时 |
| 壁钟时间 | Wall Time | 整个测试流程的实际耗时（含播放、I/O 等所有开销） |
| 音频时长 | Audio Duration | 生成的 wav 文件播放时长 |

## 典型使用流程

1. **配置模型**：在 `.env` 中设置 `TTS_MODEL=<模型名>`
2. **运行测试**：运行 `test_piper_tts.py` 或 `test_streaming_tts.py`
3. **切换模型**：修改 `.env` 中的 `TTS_MODEL`，再次运行
4. **对比结果**：运行 `compare_results.py` 生成横向对比报告
5. **批量生成**：如需大量音频素材，使用 `batch_synthesize.py`

## VS Code 运行配置

可以参考 `.vscode/launch.json` 的 VS Code 运行配置文件，其中预置了 4 个调试启动条目：

| 条目名 | 脚本 | 说明 |
|--------|------|------|
| `simple_test` | `test_streaming_tts.py` | 流式 TTS 测试，带 `--listen` 实时播放 |
| `simple_piper_tts` | `test_piper_tts.py` | Piper 模型专用测试 |
| `compare_simple_test` | `compare_results.py` | 对比 `result_piper` 下的评测结果 |
| `batch_synthesize` | `batch_synthesize.py` | 批量合成 |

在 VS Code 中按 `F5` 或从调试面板选择对应条目即可运行。
