# 功能及使用说明

本项目为**养老陪伴机器人场景下的 LLM 评测框架**，基于 [EvalScope](https://github.com/modelscope/evalscope) + OpenCompass 数据集接口，以「单项选择题（MCQ）」形式对模型在多个陪伴维度上的表现进行自动化评测，并支持多模型/多次跑次的对比。

评测维度体系共 11 大类、36 条子维度（详见 `LLM评测维度.html`），例如：智力、回复幻觉率、多轮连贯性、口语化程度、工具调用能力、记忆总结能力、ASR 错字输入鲁棒性、安全与陪伴适配性等。

## 一、目录结构

```
LLM评价标准研究/
├── 功能及使用说明.md          # 本文档
├── LLM评测维度.html           # 评测维度体系说明
├── LLM评测工具对比.html       # 评测工具选型对比
├── dataset/                   # 评测数据集（按维度分类的 JSONL）
│   └── <编号_大类>/           # 每个大类一个目录，下含若干 *.jsonl
└── eval_opencompass/          # 评测主程序目录
    ├── run_eval.py            # 主入口：批量评测
    ├── compare_results.py     # 对比 outputs/ 下各跑次，生成 HTML 对比报告
    ├── parse_results.py       # 解析已有评测结果，打印/导出答错详情
    ├── ollama_native_model.py # 自定义模型接口：Ollama 原生 /api/chat（支持关闭思考模式）
    ├── opencompass_config.py  # 自动生成，请勿手动修改
    ├── .env / .env.example    # 模型与 API 配置
    ├── datasets/              # 评测时由 dataset/ 复制过来的临时数据集
    ├── outputs/               # 评测结果输出目录
    ├── pyproject.toml        # 依赖声明（evalscope[opencompass]、opencompass）
    └── README.md
```

## 二、核心脚本功能

### 1. `run_eval.py` — 批量评测（主入口）
- 扫描 `dataset/` 下所有**以数字开头**的分类目录及其 `*.jsonl` 文件，生成为 EvalScope 的 `subset_list`。
- 通过自研的 `ollama_native` 接口调用模型，逐子集跑单项选择题评测。
- 自定义了中文陪伴机器人提示词模板，并开启 `use_cot`（链式思考）模式。
- 评测完成后：
  - 自动把结果从 `outputs/latest/` 复制为 `outputs/{MODEL_ID}_{时间戳}/`；
  - 在终端打印「按大类分组的各子维度正确率 + 总计正确率 + 答错题目详情」；
  - 将答错题目导出为 `error_details.json` 与 `error_details.txt`。
- 用法（在 `eval_opencompass/` 目录下，使用虚拟环境）：
  ```bash
  python run_eval.py
  ```

### 2. `compare_results.py` — 结果对比报告
- 扫描 `outputs/` 下所有以 `cmp_` 开头的目录（即需要对比的跑次），分别解析各子维度正确率。
- 生成一个交互式 HTML 对比报告 `outputs/comparison_report.html`：
  - 整体正确率排名表（前三名高亮）；
  - 整体正确率柱状图（Chart.js）；
  - 每个大类、每个子维度的正确率对比柱状图。
- 使用前需将待对比的结果目录**重命名为 `cmp_xxx`**（如 `cmp_gemma4`、`cmp_qwen25`）。
- 用法：
  ```bash
  python compare_results.py
  ```

### 3. `parse_results.py` — 结果解析 / 答错详情导出
- 解析指定（或最新）评测结果目录，打印各维度正确率与答错题目明细，并导出 `error_details.json/.txt`。
- 支持传入目录参数：
  ```bash
  python parse_results.py [结果目录路径]   # 不传则自动取 outputs/ 下最新目录
  ```

### 4. `ollama_native_model.py` — 自定义模型接口
- 继承 EvalScope 的 `ModelAPI`，使用 **Ollama 原生 `/api/chat` 接口**（而非 OpenAI 兼容接口）。
- 关键点：请求中设置 `think: False`，关闭 Qwen3 等模型的「思考模式」，保证输出格式稳定、便于自动化判分。
- 在 `run_eval.py` 顶部导入以触发注册（`eval_type='ollama_native'`）。

### 5. `opencompass_config.py` — 自动生成的评测配置
- 由框架自动生成，**请勿手动修改**；内容是各数据集与模型的 OpenCompass 配置。

## 三、数据集结构（`dataset/`）

- 每个**大类**对应一个子目录，目录名建议以「编号_大类」格式命名（如 `03_智力`、`04_回复幻觉率`、`09_ASR错字输入鲁棒性`、`10_安全与陪伴适配性`、`11_有帮助性`）。
- 每个子目录下是若干 `*.jsonl` 文件，每个文件对应一个**子维度**（如 `指令遵循.jsonl`、`常识推理.jsonl`、`事实正确性.jsonl`），文件字段为 `question / A / B / C / D / answer`。
- ⚠️ 注意：`run_eval.py` 会**跳过目录名不以数字开头**的分类目录。请确保所有待评测大类目录名以数字开头（如 `03_智力`），否则不会被纳入评测。

## 四、环境配置（`.env`）

复制 `.env.example` 为 `.env` 并填写：

| 配置项 | 说明 | 示例 |
|---|---|---|
| `MODEL_NAME` | 模型名称（Ollama 本地模型名，或兼容 API 模型名） | `gemma4:e2b` |
| `API_URL` | API 地址（Ollama 去掉 `/v1` 后缀走原生接口） | `http://localhost:11434/v1` |
| `API_KEY` | API Key（Ollama 本地可随意填，如 `ollama`） | `ollama` |

如需切换模型，直接修改 `.env` 中的 `MODEL_NAME` 即可（可对照 `.env` 中已注释的其他模型）。

依赖安装（首次）：在 `eval_opencompass/` 下使用 `uv`（依据 `pyproject.toml` + `uv.lock`），需 `evalscope[opencompass]>=1.8.0`、`opencompass>=0.5.2`，Python ≥ 3.10。

---

## 五、典型使用流程

1. **配置模型**：编辑 `eval_opencompass/.env`，设置 `MODEL_NAME` / `API_URL` / `API_KEY`，并确保 Ollama 服务已启动且模型已拉取。
2. **执行评测**：运行 `run_eval.py`，等待全部子维度评测完成，结果保存在 `outputs/{MODEL_ID}_{时间戳}/`。
3. **查看结果**：终端会打印正确率汇总与答错详情；也可直接运行 `parse_results.py` 重新解析/导出。
4. **多模型对比**：将两次评测的输出目录分别重命名为 `cmp_模型A`、`cmp_模型B`，再运行 `compare_results.py`，打开 `outputs/comparison_report.html` 查看对比图表。

## 六、VSCode 运行配置参考

本项目的 VSCode 调试/运行入口集中在 `launch.example.json` 中，配置了 `eval_opencompass` 虚拟环境下的两个调试条目，可直接在 VSCode 中按 F5 启动：

- **`Python Debugger: eval`** → 调试运行 `run_eval.py`（评测主程序），并设置 `justMyCode: false` 以便进入第三方库内部调试。
- **`Python Debugger: compare`** → 调试运行 `compare_results.py`（结果对比报告生成）。

如需了解这两个条目的具体参数（程序路径、解释器、工作目录等），可以**参考 `.vscode/launch.json`** 中的 VSCode 运行配置文件。
