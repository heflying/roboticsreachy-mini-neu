1. 评测工具概述

背景：针对养老陪伴机器人场景的LLM评测需求，需要对"智力"维度的4个评测项（指令遵循、中文特有知识、常识推理、逻辑推理）进行系统评测。本文档对比分析了现有的评测工具和框架，为选择合适的技术方案提供依据。

评测需求分析

评测维度 评测内容 现有数据集/工具
指令遵循 精确理解并执行复杂或模糊指令 FollowEval、IFEval
中文特有知识 理解二十四节气、传统节日、民俗习惯等 C-Eval、CMMLU
常识推理 日常生活场景的常识判断 CHARM（推荐）、CommonSenseQA
逻辑推理 简单因果、条件推理 LogiQA、FollowEval

评测框架分类

框架类型 代表工具 特点
综合评测框架 OpenCompass、lm-evaluation-harness 支持多种数据集，标准化评测流程
专项评测工具 DeepEval、AgentEvals 针对特定场景（RAG、Agent）
自定义开发 自建评测代码 高度灵活，但开发成本高

2.3 优缺点总结

推荐 OpenCompass

✅ 优势：
  • 中文支持最好：上海人工智能实验室开发，内置CHARM、C-Eval等中文数据集
  • 自定义数据集最简单：支持快速方式（直接加载CSV）和完整集成方式
  • 本地部署友好：支持HuggingFace模型本地加载，无需联网
  • 文档完善：有详细的中文教程和示例
  • 灵活评测方式：支持规则匹配、AI裁判、人工评测等多种方式
❌ 劣势：
  • 安装稍复杂（需要Python 3.10+）
  • 部分数据集需要从HuggingFace下载（可能需要科学上网）
  • 框架相对较重，学习曲线中等
⚠️ 适用场景：

中文LLM评测、需要自定义数据集、养老场景等垂直领域评测

备选 lm-evaluation-harness

✅ 优势：
  • 社区最活跃：EleutherAI维护，社区支持好
  • 安装简单：pip install即可使用
  • 支持数据集多：60+标准学术数据集
  • 轻量级：框架简洁，易于理解
❌ 劣势：
  • 中文支持较弱：主要面向英文数据集，中文数据集需要自己找
  • 自定义数据集较复杂：需要写Python类，配置不如OpenCompass直观
  • 对养老场景不够友好：主要针对学术评测，对话式评测支持不如OpenCompass
  • 文档主要是英文：对中文用户不够友好
⚠️ 适用场景：

英文模型评测、熟悉EleutherAI生态、OpenCompass装不上的备选方案

备选 DeepEval

✅ 优势：
  • 单元测试风格：对开发者友好，易于集成到CI/CD
  • 支持自定义指标：灵活定义评测标准
  • 可视化面板：提供评测结果可视化
❌ 劣势：
  • 中文支持差：主要面向英文场景，中文文档和数据集少
  • 不适合传统NLP评测：更适合RAG、Agent评测，对传统NLP任务支持不如OpenCompass
  • 社区较小：遇到问题解决可能较慢
⚠️ 适用场景：

RAG系统评测、Agent评测、需要集成到CI/CD流水线

不推荐 自定义开发

✅ 优势：
  • 完全可控：可以实现任何定制化需求
  • 无框架依赖：不依赖第三方框架
❌ 劣势：
  • 重复造轮子：数据加载、推理、评测逻辑都要自己写
  • 不可复现：评测逻辑可能写错，结果难以与其他研究对比
  • 维护成本高：每次新增模型/数据集都要改代码
  • 开发周期长：需要大量时间开发和测试
⚠️ 仅适用场景：

现有框架都满足不了需求、需要高度定制化的评测逻辑、有充足的时间和人力

3. 各框架详细说明

3.1 OpenCompass 使用指南

OpenCompass 是上海人工智能实验室开发的大模型评测框架，支持60+数据集，特别适合中文LLM评测。

安装步骤

# 1. 克隆仓库 git clone https://github.com/open-compass/opencompass.git cd opencompass # 2. 创建conda环境（推荐） conda create -n opencompass python=3.10 conda activate opencompass # 3. 安装依赖 pip install -e .

快速使用 - 自定义数据集（临时方式）

# 准备数据文件（CSV格式） # data.csv 内容： # question,answer # "北京的天气怎么样？","北京今天晴天，温度20度" # 运行评测 python run.py \ --models your_model_config \ --datasets custom \ --dataset-args '{"custom": {"path": "data.csv", "format": "csv"}}'

完整集成 - 自定义数据集（长期方式）

# 1. 创建数据集配置 # configs/datasets/your_dataset/your_dataset_gen_xx14c8.py from opencompass.datasets import GenDataset your_dataset = dict( type=GenDataset, path='data/your_dataset/data.csv', reader_cfg=dict( input_column='question', output_column='answer', ), infer_cfg=dict( prompt_template=dict( type=PromptTemplate, template=dict( begin='<用户>{question}', ), ), retriever=dict(type=ZeroRetriever), inferencer=dict(type=GenInferencer), ), eval_cfg=dict( evaluator=dict(type=AccEvaluator), pred_role='AI', ), ) # 2. 运行评测 python run.py --datasets your_dataset

养老场景示例

# 数据文件：data/elderly_intelligence/data.csv # id,dimension,sub_dimension,question,reference_answer,scoring_criteria # 1,智力,指令遵循,"请帮我设置明天早上7点的吃药提醒","好的，已为您设置明天早上7点的吃药提醒",是否正确执行指令 # 运行评测 python run.py \ --models qwen2_7b \ --datasets elderly_intelligence \ --work-dir outputs/elderly_test

3.2 lm-evaluation-harness 使用指南

lm-evaluation-harness 是EleutherAI开发的评测框架，社区活跃，支持60+数据集。

安装步骤

# 安装 pip install lm-eval # 验证安装 lm-eval --help

使用示例

# 评测本地模型 lm-eval --model hf \ --model_args pretrained=your_model_path \ --tasks hellaswag,winogrande \ --device cuda:0 \ --output_path results/ # 自定义数据集（需要写Python类） # 参考：https://github.com/EleutherAI/lm-evaluation-harness/blob/main/docs/new_task_guide.md

3.3 DeepEval 使用指南

DeepEval 是LLM评测框架，支持单元测试风格评测，适合CI/CD集成。

安装步骤

# 安装 pip install deepeval # 验证安装 deepeval --help

使用示例

from deepeval import assert_test from deepeval.test_case import LLMTestCase from deepeval.metrics import AnswerRelevancyMetric # 定义测试用例 test_case = LLMTestCase( input="请帮我设置明天早上7点的吃药提醒", actual_output="好的，已为您设置明天早上7点的吃药提醒", expected_output="好的，已为您设置明天早上7点的吃药提醒" ) # 定义评测指标 metric = AnswerRelevancyMetric(threshold=0.7) # 运行评测 assert_test(test_case, [metric])

5.2 注意事项

重要提醒：
  • 数据质量：评测数据的质量直接影响评测结果，需要精心设计测试用例
  • 评测指标：客观指标（如准确率）和主观指标（如情感陪伴质量）需要结合
  • 模型对比：需要在同一数据集上评测多个模型，才能得出客观结论
  • 持续优化：评测不是一次性工作，需要持续进行，跟踪模型优化效果

5.3 资源需求

硬件资源 GPU（推荐）、CPU、内存 评测大模型需要较高计算资源，GPU可显著加速评测
数据资源 评测数据集、测试用例 需要收集或构建养老场景评测数据
人力资源 算法工程师、数据工程师 需要熟悉OpenCompass和LLM评测的专业人员
时间资源 2-4周（完整流程） 取决于数据准备和评测复杂度

总结

核心结论：
  1. OpenCompass是最适合养老场景LLM评测的框架，中文支持好、自定义数据集简单、本地部署友好
  2. 现有数据集（CHARM、FollowEval等）可以覆盖"智力"维度的4个评测项，无需从零开始
  3. 养老场景特有维度需要自定义数据集，OpenCompass提供最简单的自定义方式
  4. 推荐混合方案：OpenCompass（主流框架） + 少量自写代码（补充）
  5. 评测是一个持续过程，需要定期运行，跟踪模型优化效果
