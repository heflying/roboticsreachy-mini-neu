# Reachy BaseBert — 功能及使用说明

## 一、项目概述

BaseBert 是一个基于中文 BERT（`chinese-bert-wwm-ext`）的 **PEFT LoRA 微调工具包**，为 Reachy 机器人构建两种文本分类模型，用于端侧/云端分流决策：

| 模型 | 输入 | 输出 | 用途 |
|------|------|------|------|
| **隐私检测** | 用户一句话 | `privacy` / `no_privacy` | 判断是否涉及敏感个人信息，涉及则送云端处理 |
| **复杂度分类** | 用户一句话 | `simple` / `complex` | 判断句子复杂度是否超出端侧小模型处理能力 |

隐私判断依据 **GB/T 45574-2025**《数据安全技术 敏感个人信息处理安全要求》，覆盖生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹、通信内容、个人身份信息（PII）、未成年人信息、健康监测等 10 大类敏感信息。

## 三、脚本功能详解

### 1. `train_lora.py` — 主训练脚本

使用 PEFT LoRA 对 BERT 进行序列分类微调。

**核心特性**：
- LoRA 微调（r=8, alpha=32, target_modules=["query","value"]）
- 自动解冻 classifier + pooler 层
- 类别不平衡自动加权（逆频率权重）
- 每 N epoch 保存合并模型检查点
- 每 N epoch 评估并追踪最佳 F1
- TensorBoard 训练可视化

**关键参数**：

| 参数 | 说明 |
|------|------|
| `--train_csv` | 训练 CSV（支持多个） |
| `--val_csv` | 验证 CSV |
| `--label_list` | 标签列表，逗号分隔，如 `"no_privacy,privacy"` |
| `--from_model_dir` | 预训练模型目录 |
| `--output_model_dir` | 输出目录 |
| `--epochs` | 训练轮数 |
| `--batch_size` | 批次大小 |
| `--save_every_epochs` | 每 N epoch 保存检查点 |
| `--do_train` | 启用训练模式 |
| `--do_eval` | 启用评估模式 |

### 2. `test_model.py` — 批量评估脚本

对 CSV 测试文件进行评估，计算指标并生成 HTML 报告。

**输出内容**：准确率、错误率、F1(macro)、各类精确率/召回率/F1、错误样本表格、汇总表格。

**关键参数**：

| 参数 | 说明 |
|------|------|
| `--eval_model_dir` | 模型目录 |
| `--test_csv` | 测试 CSV（支持多个） |
| `--recall_label` | 重点关注的召回标签，如 `privacy` |
| `--output_html` | HTML 报告路径（默认 `test_report.html`） |

### 3. `test_direct_output.py` — 轻量推理脚本

加载模型对输入句子进行推理，在命令行直接输出每个类别的概率，无需 HTML 报告。

**关键参数**：

| 参数 | 说明 |
|------|------|
| `--eval_model_dir` | 模型目录 |
| `--sentence` | 单条句子（可重复） |
| `--input_file` | 文本文件（每行一句）或 CSV |

### 4. `generate_train_dataset.py` — 合成训练数据

使用 LLM 根据 GB/T 45574-2025 标准生成隐私/非隐私训练数据。覆盖 10 种隐私类别 + 5 种非隐私类别。

### 5. `augment_data.py` — 数据增强

支持三种增强方法：EDA（同义词替换/随机插入/交换/删除）、回译（中→英→中）、LLM 释义。

### 6. `llm_label.py` — LLM 辅助标注

使用 LLM 对 CSV 数据自动标注隐私标签，支持 Ollama/Qwen/Spark 三种后端，支持断点续传。

### 7. `review_labels.py` — 交互式标签审核

命令行逐条审核工具，单键操作（`↑`/`p` 上一条，`n` 翻转标签，`Space`/`Enter` 确认，`q` 保存退出），修改前自动创建 `.bak` 备份。

### 8. `convert_20ng.py` — 20 Newsgroups 转换

将 20 Newsgroups PII-Augmented 数据集（英文 JSONL）转换为训练 CSV：NLTK 分句 → 滑动窗口采样 → 实体标签判断 → LLM 翻译为中文。

### 9. `convert_interaction_dialogue.py` — 对话数据转换

将 Interaction_Dialogue_with_Privacy 数据集（中文 JSON）转换为训练 CSV：提取 user/assistant 话语 → 隐私短语匹配 → 输出（无需翻译）。

### 10. `convert_privacy_care.py` — Privacy-Care 转换

将 privacy-care-interactions 数据集（英文 JSONL）转换为训练 CSV：说话人分割 → 标签判断（category==2→privacy）→ LLM 翻译为中文。

### 11. `llm_client.py` — 公共 LLM 客户端

统一的多后端 LLM 调用接口，支持 Ollama、Qwen（阿里云百炼）、Spark（讯飞星火），通过 `.env` 文件配置密钥。

## 四、完整使用流程

### Step 1: 准备数据

```bash
# 选项 A：转换第三方数据集
python scripts/convert_20ng.py --llm_backend ollama
python scripts/convert_interaction_dialogue.py
python scripts/convert_privacy_care.py --llm_backend ollama

# 选项 B：生成合成数据
python scripts/generate_train_dataset.py --llm_backend ollama --num_per_category 50
```

### Step 2: 数据增强（可选）

```bash
python scripts/augment_data.py --input data/train.csv --output data/train_augmented.csv \
    --augment_times 2 --use_eda --use_paraphrase --llm_backend ollama
```

### Step 3: LLM 辅助标注（可选）

```bash
python scripts/llm_label.py --input data/train.csv --llm_backend ollama
```

### Step 4: 人工审核标签

```bash
python scripts/review_labels.py --input data/train.csv
```

### Step 5: 训练模型

```bash
python scripts/train_lora.py --do_train \
    --train_csv data/privacy/train.csv \
    --val_csv data/privacy/val.csv \
    --from_model_dir origin_model/chinese-bert-wwm-ext \
    --output_model_dir output_model/privacy \
    --label_list "no_privacy,privacy" \
    --epochs 20 --batch_size 2
```

### Step 6: 测试模型

```bash
python scripts/test_model.py \
    --eval_model_dir output_model/privacy/latest \
    --test_csv data/privacy/test.csv \
    --recall_label privacy \
    --output_html test_report.html
```

## 七、数据格式

所有 CSV 文件统一格式：**两列 `text,label`**，带表头，UTF-8 编码。

示例：
```csv
text,label
"我的银行卡号是6222021234567890",privacy
"今天天气真好",no_privacy
```

## 八、VSCode 运行配置

项目提供了 `.vscode/launch.json` 文件，包含 10 个预配置的调试/运行条目，可在 VSCode 中按 `F5` 直接运行，无需手动输入命令行参数。条目分类如下：

### 隐私检测（5 个）

| 条目名 | 功能 |
|--------|------|
| `new_privacy_train` | 从头训练隐私模型（从原始 BERT 开始，6 个训练集，20 epochs） |
| `continue_privacy_train` | 断点续训隐私模型（从 `output_model/privacy/latest` 继续） |
| `privacy_eval` | 仅评估隐私模型（不做训练，纯 `--do_eval`） |
| `privacy_test` | 批量测试隐私模型（6 个测试集，生成 `test_report.html`） |
| `privacy_test_direct` | 轻量推理测试（命令行逐条输出概率） |

### 复杂度分类（4 个）

| 条目名 | 功能 |
|--------|------|
| `new_complex_train` | 从头训练复杂度模型（100 epochs） |
| `continue_complex_train` | 断点续训复杂度模型 |
| `complex_eval` | 仅评估复杂度模型 |
| `complex_test` | 测试复杂度模型（对 `test_sentences.txt` 推理） |

### 数据处理（2 个）

| 条目名 | 功能 |
|--------|------|
| `llm_label` | LLM 自动标注隐私数据 |
| `review_label` | 交互式人工审核标签 |

> **提示**：以上条目涵盖了隐私检测和复杂度分类两个任务的训练、评估、测试全流程，以及数据标注和审核的辅助工具。你可以直接在 VSCode 的「运行和调试」面板中选择对应条目执行，无需手动输入命令行。如需调整参数（如数据集路径、epoch 数等），可编辑 `launch.json` 中对应条目的 `args` 字段。
